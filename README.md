# Artificial-Intelligence-And-Open-Science-In-Research-Software-Engineering
This repository performs a text analysis over 10 open-access research papers
using Grobid to extract structured information from PDF documents.

We selected 10 open-access papers from arXiv related to distributed
computing and data-intensive systems.

1. PDFs are processed using Grobid.
2. Grobid generates TEI XML files.
3. XML files are analyzed to extract:
   - Abstract text
   - Figures
   - Links

## Requirements
Python libraries:
- beautifulsoup4
- matplotlib
- wordcloud
- requests

Grobid image in docker: https://hub.docker.com/r/grobid/grobid/tags



## Reproducibility

1. Run Grobid with Docker:
docker run -t --rm -p 8070:8070 grobid/grobid:${latest_grobid_version}-crf

2. Process the PDFs:
python scripts/run_grobid.py

3. Generate the analysis:

python grobidClient
python wordCloud.py
python countFigures.py
python extract_links.py



## Validation
### Grobid Extraction

We validated that Grobid correctly parsed the PDF documents by
inspecting the generated TEI XML files.

We verified that the following sections were correctly extracted:
- abstract
- figures
- references
- links

These elements were consistent with the structure of the original PDF
documents.


### Word Cloud

To validate the word cloud generation, we manually inspected several XML files
generated by Grobid and compared the extracted abstract text with the
original PDF documents.

We verified that:
- The abstract section was correctly extracted from the `<abstract>` tag.
- The most frequent words appearing in the word cloud (e.g., "data",
  "performance", "computation") are consistent with the topics discussed
  in the abstracts.

This confirms that the word cloud correctly represents the dominant
terms present in the dataset.
